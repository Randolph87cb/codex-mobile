# 会话详情页 UI 说明

本文档只聚焦 Android 会话详情页当前的 UI 结构、截图验证方式，以及 showcase 截图与真实线程页之间的边界。

## 当前已落地的 UI 调整

- 顶部状态条已经压缩成紧凑四列指标。
- 顶栏右侧新增全局额度指示器，不再在正文区单独放一张大额度卡。
- 目标卡默认态优先展示 `目标 + 状态 + 一行正文`，展开后再看完整信息。
- 待发送图片区使用固定尺寸预览窗，不再按原图比例把托盘整体拉高。
- 点击待发送图片缩略图后，仍然查看原图。
- `你 / Codex` 的真实聊天正文位于左右头像槽之间，不再侵占对方头像下方空间。
- 普通消息气泡按内容自然撑开，并通过最大宽度限制长文本；短消息不会默认铺满整行。
- 普通消息不再常驻复制按钮，长按气泡后弹出菜单，菜单提供“复制”和“选择文本”。
- `Codex` 气泡使用纯色浅蓝，用户气泡使用纯色淡粉；系统和工具消息使用中性浅灰。
- 设置页提供全局字体大小选项：小、标准、大。
- 连接层异常不再插入正文消息流，而是通过顶部状态区内的轻通知条提示当前连接状态。

## 当前实时流恢复规则

- 顶部状态区负责承载连接层提示，例如：
  - 实时流已断开，准备重连
  - bridge 正在重启
  - 当前停留在最后一次收到的内容快照
- `Broken pipe`、`Software caused connection abort` 这类传输层错误不再作为系统消息气泡写进 transcript。
- 只有会话语义事件，例如审批请求、审批结果、明确的任务中断，才继续进入 transcript 或结构化执行区。
- 当详情页实时流断开后重新收到 `session.started`：
  - 客户端会先恢复实时流连接状态；
  - 随后主动补一次 `GET /api/session/:id` 详情快照；
  - 目的是把断线窗口里服务端已经产出、但客户端没来得及收到的正文补回来。
- 应用从后台回到前台时，如果当前仍停留在详情页：
  - 即使实时流表面上仍是已连接，也会先主动补一次 `GET /api/session/:id`；
  - 目的是覆盖“连接没有及时报错，但详情页内容已经落后”的情况。
- Android 端现在用显式的“待补快照”状态来绑定这次追平：
  - 断线、实时流监听异常、前台恢复触发重连时，先记录当前 `sessionId` 需要补拉；
  - 之后该会话重新收到 `session.started` 时，再稳定执行一次详情快照补拉；
  - 不再依赖瞬时连接状态或提示文案去猜测“这是不是一次重连恢复”。
- 如果重连后没有新的 `assistant.delta`，这次补快照仍然是正文追平的关键路径；不能只依赖“实时流重新连接成功”这一个状态。

## 当前全局额度交互

详情页全局额度不再占用正文区块，当前规则如下：

- 位置：顶栏右侧，位于刷新按钮左侧。
- 形态：两个小圆点。
  - 左点表示 `5 小时`
  - 右点表示 `1 周`
- 颜色只表达大致使用区间，不直接写数值：
  - 低占用：绿色
  - 中占用：黄色
  - 偏高：橙色
  - 接近打满：红色
  - 暂无数据：空心灰点
- 点击后展开小浮层，显示：
  - `5 小时` 剩余百分比
  - `1 周` 剩余百分比
  - 已用百分比
  - 重置时间
- 如果额度刷新失败，但本地仍有上次成功快照：
  - 继续显示旧快照
  - 浮层底部附加错误提示，说明当前显示的是上次同步结果

这个交互的目的不是给出精确监控面板，而是把“额度健康度”压缩进顶栏，不挤占真实对话空间。

## 当前会话消息布局规则

详情页真实消息采用固定三栏结构：

- 左头像槽：固定 `40dp`。
- 中间消息槽：使用剩余宽度，消息气泡只能出现在这里。
- 右头像槽：固定 `40dp`。

Codex 消息显示在左头像右侧，用户消息显示在右头像左侧。没有头像的一侧仍保留空槽，所以任一方向的消息都不会进入对方头像下方。

气泡宽度规则：

- 气泡默认按内容自然宽度显示。
- 长文本最多扩展到 `ConversationBubbleMaxWidth`。
- Markdown 文本块在消息气泡内关闭强制 `fillMaxWidth()`，在其他场景仍保持默认填满父容器。

当前颜色 token：

- Codex 头像：`#2D4B73`
- Codex 气泡：`#EAF1F8`
- Codex 气泡边框：`#D5E0EA`
- 用户头像：保持当前深红 token
- 用户气泡：`#FFF1F3`
- 用户气泡边框：`#F3D4D9`
- 系统 / 工具气泡：`#F3F4F5`
- 代码块背景：`#F8FAFC`

复制交互：

- 默认状态下，消息正文不启用 `SelectionContainer`，避免长按优先进入系统文本选择。
- 长按气泡弹出深色菜单，提供“复制”和“选择文本”。
- 点“复制”复制整条消息。
- 点“选择文本”后，当前气泡正文进入统一选择态，再使用系统选择菜单复制局部文本。
- 同一条消息内，普通段落、列表、引用和代码块共享同一个选择域；不再只能复制单个段落。
- 代码块仍保留独立复制入口，但在“选择文本”模式下也并入同一条消息的统一选择域。
- 进入选择态后，点击输入框、消息外区域、执行普通“复制”，或收起当前消息，都会退出选择态。

## 重要边界

showcase 截图不是“真机真实工作线程截图”。

原因很直接：

- showcase 吃的是 `SessionDetailScreenshotTest.kt` 里手工构造的标题、目标、短消息样本和固定附件集。
- 真实手机线程页直接吃 bridge 返回的真实 `transcript`、执行过程、目标正文和待发送附件。

这意味着：

- showcase 越来越像参考稿，只能证明“样板页结构更像”；
- 它不能自动证明“你手机上正在工作的那条真实线程也会一样像”。

## 后续如果目标是“真机真实线程也像参考图”

不要只抠 showcase 截图，还要优先检查这些真实页面策略：

- 消息正文宽度是否足够
- 长回复是否需要默认折叠
- 系统 / 执行过程卡是否需要默认摘要化
- 目标卡默认态是否还需要继续压缩
- 用户那条线程里是否真的存在待发送图片区，而不是只有历史消息

## 真实可读性相关的调整

旧版真实消息宽度曾经使用百分比参数：

- 用户消息约 `74%`
- Codex 消息约 `80%`

最新实现已经不再依赖这组百分比。当前以固定左右头像槽加中间消息槽为准，消息气泡在中间槽内按内容自然宽度和最大宽度共同决定尺寸。
