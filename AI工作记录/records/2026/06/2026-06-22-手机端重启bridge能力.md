# 2026-06-22 手机端重启 bridge 能力

## 目标

- Android 手机端提供最小入口触发 Windows bridge 重启。
- bridge 收到请求后不在当前请求内同步杀自身，而是进入 drain、广播 lifecycle、启动独立后台 helper，并立即返回 202。
- helper 延迟约 1 秒后执行 `scripts/restart-bridge-background.ps1`。

## 改动

- bridge 新增 `/api/admin/restart`：
  - 复用现有 `/api/*` bearer auth 中间件。
  - 调用 `lifecycle.beginDrain()` 并 `broadcastLifecycle()`。
  - 通过可注入 `BridgeRestartScheduler` 调度重启，测试中使用 fake scheduler。
- bridge 新增 `restart-scheduler.ts`：
  - 使用 detached/hidden `powershell.exe` helper。
  - helper 内 `Start-Sleep -Milliseconds 1000` 后调用仓库脚本 `scripts/restart-bridge-background.ps1`。
  - 不传 `-SkipBuild`，保留重启脚本默认构建流程，确保源码更新后手机端触发重启能加载新的 production bundle。
- Android 数据层新增 `restartBridge()`，真实实现调用 `POST /api/admin/restart`。
- Android 设置页连接管理区域新增“重启 Bridge”按钮。
- ViewModel 新增 `restartBridge()`，请求成功后提示正在重启，并轮询恢复连接、刷新会话列表和额度；若原本有实时流活动，恢复后重新建立实时流。

## 2026-06-22 后续修复：重启后手机端提示残留

### 目标

- 修复手机端点击“重启 Bridge”后，进入线程详情页长期显示“bridge 即将重启 / 正在重启”的问题。
- 按用户要求使用 harness + subagent 结构执行；subagent 使用 `fork_context=false`，不提交 Git，主线程负责复核、验证、提交和推送。

### 根因

- Android `connect()` 只按 `/health` HTTP 200 判定连接恢复，没有识别旧 bridge 的 `lifecycle.draining=true` / `phase=restarting`。
- ViewModel 重启恢复成功后只在部分分支清理 `bridgeRestartExpectedSessionId` 和实时流 notice，导致旧 restart `fallbackNotice` / `lastEventText` 可残留到后续线程详情。
- Bridge `/api/admin/restart` 若调度 helper 失败，已进入的 drain 状态没有撤销。

### 改动

- `RealBridgeDataProvider.connect()` 读取 `/health.lifecycle`；draining 或 restarting 时抛出“bridge 正在重启，请稍后再试。”，让恢复轮询继续等待。
- `AppViewModel` 新增集中清理重启恢复状态的 helper，恢复成功、恢复失败、`StreamOpened`、`SessionStarted`、非 restarting lifecycle 事件都会清理 pending recovery/reconnect 状态和旧重启 notice。
- `BridgeLifecycle` running 事件会清掉 restart 横幅；打开其他线程时不会继承旧 restart notice。
- Bridge lifecycle controller 新增 `cancelDrain()`；admin restart 调度失败时撤销 drain，并在错误响应及 `/health` 中体现 running。

### 测试

- Android 新增测试：
  - health draining 不算连接成功。
  - restart recovery 成功后清理 restart fallback notice。
  - lifecycle running 清理 restart notice。
  - 打开其他线程不残留旧 restart notice。
- Bridge 测试补充 scheduler failure 后 lifecycle 回到 running。
- 验证命令：
  - `cd android; .\gradlew.bat testDebugUnitTest`：通过。
  - `powershell -ExecutionPolicy Bypass -File .\scripts\build-android-debug.ps1`：通过。
  - `cd bridge; npm run check`：通过。
  - `cd bridge; npm test`：通过。
