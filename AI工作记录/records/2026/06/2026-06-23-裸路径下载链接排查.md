# 2026-06-23 裸路径下载链接排查

## 背景

- 用户反馈发送到手机端的 Windows 本地路径没有直接变成下载链接。
- 示例路径：
  - `D:\workspace\codex-pet-suite\pets\tuantuan\workspace\qa\behavior-contact-sheet-full.png`
  - `D:\workspace\codex-pet-suite\pets\tuantuan\workspace\qa\runtime-preview-board.png`

## 排查结论

- bridge 已提供 `/api/file/download?path=...` 和 `/api/image/file?path=...`，且日志中已有 `codex-pet-suite` 路径下载成功的 200 记录。
- 两个示例文件在本机均存在。
- Android 侧 `resolveTranscriptFileDownloadRequest` 可以处理 `bridge-file://...`、Markdown 链接目标里的 Windows 绝对路径和相对路径。
- 问题点在消息渲染入口：`agentMessage.text` 会被原样放入 transcript，Android Markdown 渲染只为 `[label](target)` 形式生成点击注解，裸 Windows 路径不会自动 linkify。

## 后续建议

- 若要修复，应在 Android Markdown inline 解析层增加本地路径自动链接化，或在 bridge transcript 生成层把裸本地路径转换成 `bridge-file://...` Markdown 链接。
- 需要同时补 Android 单元测试，覆盖裸 Windows 绝对路径、多行路径、反斜杠路径和代码块/行内代码不误识别。
