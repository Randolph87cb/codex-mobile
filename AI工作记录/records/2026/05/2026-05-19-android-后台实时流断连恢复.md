## 任务输入摘要

- 最终结果：解决 Android 端切到后台后详情页实时流容易中断且回到前台后不自动恢复的问题。
- 现有素材：用户提供的 Android 日志、`AppViewModel.kt`、`CodexMobileApp.kt`、`RealBridgeDataProvider.kt`、`AppViewModelTest.kt`。
- 明确约束：保持 Android 轻客户端定位；优先修复自动恢复链路，不默认引入前台服务或长期后台保活；改动后补 Android 侧测试。
- 完成标准：后台导致 WebSocket 断开后，前台内自动重连；应用回到前台时会补连详情页实时流；补单测覆盖前台断线重连和后台后前台恢复。
- 产出后动作：执行 Android 构建与单测，更新记录，检查 Git 状态并按规则提交推送。

## 关键判断

- 日志显示 `2026-05-19 03:57:53` WebSocket 因 `Software caused connection abort` 断开，但随后 bridge 的 HTTP 会话详情和会话列表请求仍能恢复成功，说明核心问题不是 bridge 进程退出，而是 Android 侧缺少实时流重连策略。
- 现有实现只在 `openSessionDetail()` 时建立一次实时流；`onFailure` 和 `StreamClosed` 之后只做 HTTP 快照回填，不会自动重连，也不会在 Activity 回到前台时补连。
- 在当前项目“轻客户端”约束下，本次不做前台服务保活；修复目标收敛为“断线自动恢复”和“前台补连”。如果后续需要后台持续实时收流，再单独评估前台服务。

## 实际改动

- `android/app/src/main/java/com/openai/codexmobile/AppViewModel.kt`
  - 增加实时流重连状态和前后台感知。
  - 对 `StreamClosed` 与 WebSocket 监听异常增加自动重连调度。
  - 应用回到前台时，如果详情页实时流处于断开状态，会立即补连。
- `android/app/src/main/java/com/openai/codexmobile/ui/CodexMobileApp.kt`
  - 监听 Activity 生命周期的 `ON_START` / `ON_STOP`，把前后台切换通知给 `AppViewModel`。
- `android/app/src/main/java/com/openai/codexmobile/data/RealBridgeDataProvider.kt`
  - 为 OkHttp WebSocket 增加 `20s` ping keepalive，降低空闲链路被中间网络设备回收的概率。
- `android/app/src/test/java/com/openai/codexmobile/AppViewModelTest.kt`
  - 新增“前台断线自动重连”和“后台不重连、回前台再补连”两条测试。

## 追加调查：权限开关现状

- 用户继续询问“是否还只能沙箱模式，能不能给最高权限”。
- 当前结论：
  - Android 与 bridge 公开出来的只有 `approvalMode`，也就是“手动审批 / 自动审批”。
  - `bridge/src/app-server-runner.ts` 当前只把 `approvalMode` 映射成 `approvalPolicy`，没有把 `sandbox` 或 `permissionProfile` 传给 `thread/start`、`thread/resume`、`turn/start`。
  - `docs/generated/` 的协议生成物里已经能看到 `sandbox`、`danger-full-access`、`permissionProfile` 等字段，说明上游协议层具备能力，但本项目 bridge / Android 还没接。
- 影响：
  - 现在手机端不能显式切到 `danger-full-access`。
  - 把 `approvalMode` 切成 `auto` 也不等于“最高权限”；它只是不再人工确认审批请求。
- 后续如果要补“最高权限”开关，至少要同时改：
  - `bridge/src/types.ts`
  - `bridge/src/app.ts`
  - `bridge/src/app-server-runner.ts`
  - `android/app/src/main/java/com/openai/codexmobile/data/BridgeApi.kt`
  - `android/app/src/main/java/com/openai/codexmobile/data/RealBridgeDataProvider.kt`
  - `android/app/src/main/java/com/openai/codexmobile/AppViewModel.kt`

## 追加实施：接通 sandboxMode

- 用户随后明确要求把权限模式接到手机端。
- 本次实施范围：
  - bridge 新增 `sandboxMode` 字段，支持 `read-only / workspace-write / danger-full-access`。
  - Android 设置页与会话详情配置页新增“文件权限”切换。
  - bridge 把 `sandboxMode` 传给 `thread/start`、`thread/resume`，并在 `turn/start` 上映射成对应 `sandboxPolicy`。
  - Android 数据层、实时流事件、设置存储、ViewModel、测试桩同步补齐。
