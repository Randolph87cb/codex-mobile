## 任务输入摘要

- 最终结果：为 bridge 增加最小可用安全控制面，覆盖 token 认证和 `cwd` 目录白名单。
- 现有素材：`bridge/src/app.ts`、`bridge/src/types.ts`、`bridge/src/session-store.ts`、`bridge/tests/app.test.ts`、`docs/api.md`、`README.md`、`AGENTS.md`。
- 明确约束：只做 bridge 安全控制面，不做审批闭环，不做 Android WebSocket；优先环境变量配置；尽量保持 API 兼容；`/health` 保持公开。
- 完成标准：bridge 具备可配置 token 鉴权与 `cwd` 白名单、明确错误语义、自动化测试和必要文档更新。
- 产出后动作：运行 `bridge` 校验与测试，并确认 Android MVP 影响范围。

## 方案决策

- 采用两个环境变量：
  - `CODEX_MOBILE_AUTH_TOKEN`
  - `CODEX_MOBILE_ALLOWED_CWDS`
- `/health` 继续公开，作为局域网/Tailscale 诊断入口。
- 其余 `/api/*` 仅在配置 token 后才要求 `Authorization: Bearer <token>`，避免默认破坏当前 Android MVP。
- `cwd` 白名单仅在创建会话时校验；未配置白名单时不限制目录，也保留当前 Android 默认相对路径请求体行为。
- 一旦启用白名单，`cwd` 必须是绝对路径且位于允许目录内，避免相对路径带来的边界歧义。

## 风险与影响

- 当前 Android MVP 尚未支持附带 Bearer token；启用 token 后，Android 现有版本除 `/health` 外的 `/api/*` 调用会收到 `401`。
- 默认不配置安全项时，Android 当前连接方式不受影响。
