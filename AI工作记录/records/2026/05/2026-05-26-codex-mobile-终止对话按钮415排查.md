## 背景

详情页中的“终止对话/中断当前轮”按钮点击后没有停止当前会话，前端 toast 已明确显示 bridge 返回 `415 Unsupported Media Type`，需要区分是 UI 绑定问题、Android 请求构造问题，还是 bridge 路由处理问题。

## 关键过程

- 先检查 Android 调用链，确认 `SessionDetailScreen` 的中断按钮绑定到 `AppViewModel.interruptSelectedSession()`。
- 继续检查 `AppViewModel`，确认按钮点击后会调用 `BridgeApi.interruptSession(sessionId)`，UI 层不是无效点击。
- 检查 `RealBridgeDataProvider.interruptSession()`，发现它向 `POST /api/session/:id/interrupt` 发请求，但没有显式传 body。
- 检查 `bridge/src/app.ts`，确认 `/api/session/:id/interrupt` 路由本身不读取 body，只做 `resolveSessionRecord()` 后调用 `runner.interrupt(session.id)`。
- 查看 `.logs/bridge/bridge-stdout.log`，确认 2026-05-26 22:26 左右两次 `POST /api/session/019e643b-cdd3-7182-ac96-cf4de9fdf0fe/interrupt` 都返回 `415`，错误码为 `FST_ERR_CTP_INVALID_MEDIA_TYPE`，异常栈停在 Fastify `content-type-parser`，说明请求在进入路由前就被拒绝。
- bridge 测试 `bridge/tests/app.test.ts` 中对 `/interrupt` 的 `app.inject()` 是通过的，说明路由逻辑本身没坏，问题更像是真机 Android 发出的请求格式与 Fastify 期望不兼容。

## 结果

- 已确认根因方向：
  Android 端“中断”请求是一个无显式 body 的 `POST`；真机实际发出的请求被 Fastify 当作不支持的媒体类型处理，因此在 bridge 路由执行前直接返回 `415`。
- 已按最小修复方案完成修改：
  - `android/app/src/main/java/com/openai/codexmobile/data/RealBridgeDataProvider.kt`
    - `interruptSession()` 现在显式发送 `{}` JSON body，确保请求稳定带 `application/json; charset=utf-8`。
  - `android/app/src/test/java/com/openai/codexmobile/data/RealBridgeDataProviderTest.kt`
    - 新增回归测试，使用 `MockWebServer` 断言 `/interrupt` 请求路径、`Content-Type` 和 body。
  - `android/app/build.gradle.kts`
    - 新增单测依赖 `com.squareup.okhttp3:mockwebserver:4.12.0`。
  - 文档同步：
    - `docs/api.md`：补充 `/api/session/:id/interrupt` 的空 JSON body 与 `Content-Type` 约定，并记录 415 兼容性说明。
    - `android/README.md`：补充 Android 侧中断接口接入注意事项。
    - `README.md`：补充“终止当前轮”动作当前使用显式空 JSON body 的高层说明。
- 已确认不是以下问题：
  - 不是按钮点击没绑定。
  - 不是 `AppViewModel` 没调用中断接口。
  - 不是 `runner.interrupt()` 逻辑报错。
- 已完成验证：
  - 源码链路检查：`SessionDetailScreen.kt` -> `AppViewModel.kt` -> `RealBridgeDataProvider.kt` -> `bridge/src/app.ts`
  - 日志核对：`.logs/bridge/bridge-stdout.log`
  - 现有测试核对：`bridge/tests/app.test.ts`
  - Android 单测：`cd android; .\gradlew.bat testDebugUnitTest`
  - Android 构建：`powershell -ExecutionPolicy Bypass -File .\scripts\build-android-debug.ps1`
  - 首次验证前因 Kotlin 增量缓存冲突失败，已通过 `.\gradlew.bat --stop` 并清理 `android/app/build/kotlin` 后重跑通过。
  - 文档一致性检查：确认 `docs/api.md`、`android/README.md`、`README.md` 三处都已同步写明 `/interrupt` 的显式空 JSON 调用约定。

## 可复用经验

- 对移动端按钮“没生效”的问题，优先分三层排查：UI 绑定、客户端请求构造、bridge/服务端网关层。
- Fastify 返回 `FST_ERR_CTP_INVALID_MEDIA_TYPE` 时，优先怀疑请求头或空 body `POST` 的兼容性，而不是业务 handler。
- 对无请求体的 `POST` 接口，Android 和 bridge 最好统一约定为发送 `{}` JSON，避免不同 HTTP 实现对空 body 的默认行为不一致。
