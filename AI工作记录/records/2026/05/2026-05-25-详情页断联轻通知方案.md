# 详情页断联轻通知方案

- 日期：2026-05-25
- 目标：分析对话详情页因切后台或其他断联导致错误消息落入消息流的问题，设计更轻的通知方案。
- 当前阶段：只读分析。
- 约束：先不给出实质代码修改；保持 Android 端轻客户端；用户可见文案默认中文。

## 当前发现

- 断联噪音进入对话流的直接来源在 `android/app/src/main/java/com/openai/codexmobile/AppViewModel.kt`。
- `SessionStreamEvent.Error` 的非重试分支会调用 `appendSystemMessage(detail, "系统：${event.message}", ...)`，把 `Broken pipe`、`Software caused connection abort` 之类错误追加进 transcript。
- `RealBridgeDataProvider.observeSessionEvents()` 的 WebSocket `onFailure()` 当前会发送 `SessionStreamEvent.Error`，随后以异常关闭 flow；`AppViewModel.startSessionStream()` 的 catch 又会再走一轮重连/回退逻辑，因此同一次连接层失败同时产生“对话气泡污染”和“状态区回退”两套反馈。
- `SessionRealtimeUiState` 已经有 `connectionText`、`lastEventText`、`fallbackNotice` 三个适合轻通知的字段，但 `SessionDetailScreen` 目前只在 `StatusStrip` 里用到了 `isConnected`，没有把这些说明信息渲染出来。
- `mergeSessionDetail()` 会保留更“完整”的 transcript，因此一旦把错误追加进去，即使后续刷新快照恢复，错误文本也可能继续留在会话内容里。

## 修改方向草案

- 只把“会话语义事件”留在 transcript：
  - 保留审批请求、审批结果、明确的任务中断等可追溯事件。
  - 移除连接层/传输层故障写入 transcript 的路径。
- 连接层故障改走轻通知：
  - 优先复用 `fallbackNotice` + `connectionText`。
  - 在详情页 `StatusStrip` 下方或内部增加一条轻量 inline notice，而不是插入新的消息气泡。
- 更干净的事件归类：
  - WebSocket `onFailure()` 导致的 socket 中断更适合映射成 `StreamClosed` / 连接中断语义，而不是 transcript 级 `Error`。
  - bridge 主动推送的结构化 error 事件仍保留 `Error` 语义，但默认也只做状态通知，不直接写入 transcript。

## 待确认方案

- 倾向采用：顶部状态区下方增加一条可自动消失或可覆盖更新的轻通知横条，文案类似“实时流已断开，当前显示最后一次快照；回到前台后会自动重连”。
- 倾向不采用：继续把这类错误做成系统消息气泡。

## 后续排查补充

- 用户反馈：详情页显示“实时流已重连”，但消息正文没有更新。
- 当前定位：
  - 断流时 `AppViewModel` 只会在 `StreamClosed` 分支按条件调用一次 `refreshSessionSnapshot(sessionId)`。
  - 重新连上后，`SessionStarted` 分支只更新 `cwd / model / status / goal / pendingApproval` 等元数据，不会再主动刷新详情快照。
  - `session.started` 实时事件本身也不携带 transcript 内容，因此如果断开窗口里的回复已经在服务端完成，而重连后又没有新的 `assistant.delta`，客户端就会显示“已重连”但正文仍停留在旧快照。
- 结论：这是“重连后缺少补快照”问题，不是轻通知改造本身导致的展示文案问题。

## 重连补快照修复

- 已修复：
  - `android/app/src/main/java/com/openai/codexmobile/AppViewModel.kt`
  - 当实时流从“已断开/准备重连”恢复到 `session.started` 时，客户端会主动补一次 `refreshSessionSnapshot(sessionId)`。
  - 首次打开详情页的“正在连接实时流”不触发这次额外补拉，避免无意义重复请求。
- 已补测试：
  - `android/app/src/test/java/com/openai/codexmobile/AppViewModelTest.kt`
  - 覆盖“重连成功但没有新 delta，仍能补回断线期间正文”的场景。
